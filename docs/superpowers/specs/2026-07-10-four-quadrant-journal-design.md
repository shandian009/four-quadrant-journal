# 四象日志 Windows 桌面版产品设计规格

日期：2026-07-10  
状态：待用户最终审阅  
目标平台：Windows 10/11 64 位

## 1. 产品定位

“四象日志”是一款纯本地、无需账号的 Windows 桌面效率软件。它把四象限任务管理、月历、每日复盘和基础提醒放在同一个工作流中，帮助上班族每天完成三件事：确定优先级、执行与记录、回顾并安排明天。

产品视觉记忆点是固定的“今日工作台”布局和周一至周五五套情绪皮肤。皮肤随工作日自动切换，但只改变色彩令牌，不改变页面布局、功能位置或数据。

## 2. 已确认的产品约束

- 第一版只支持 Windows 桌面端。
- 所有数据只保存在本机，不注册、不登录、不依赖云服务。
- 支持 Windows 到时通知和稍后提醒。
- 核心是“任务 + 每日复盘”，不是纯日记，也不是单纯的操作记录。
- 主界面沿用已确认的工作台布局：左侧导航；上方任务、日历、统计；下方复盘。
- 周一至周五使用五套独立皮肤；皮肤不能改变排版结构。

## 3. 技术路线选择

### 方案比较

1. Electron + React + TypeScript + SQLite：界面还原度高，Windows 通知、托盘和自动化测试成熟；代价是安装包较大。
2. Tauri + React + SQLite：安装包更小，但需要 Rust 工具链，当前构建和维护成本明显更高。
3. Python + PySide6 + SQLite：本地逻辑简单，但高级界面还原、前端动效和持续 UI 测试成本更高。

### 最终选择

采用 Electron + React + TypeScript + SQLite。Node.js 和构建工具只用于开发与打包；最终用户安装 EXE 后不需要安装 Node.js。

## 4. 第一版功能范围

### 4.1 今日工作台

- 展示当前日期、星期和当天皮肤名称。
- “今日优先事项”按优先级与截止时间排序，显示最多五项，支持进入完整列表。
- 月历显示有事项、已完成、逾期或已写复盘的日期状态。
- “今日概览”展示计划完成度、专注时长、完成事项和待处理事项。
- “今日复盘”包含今日收获、待改进、明日重点三个输入区，自动保存草稿。
- 快捷按钮可在任何页面添加事项。

### 4.2 四象限任务

- 四个象限固定为：重要且紧急、重要不紧急、紧急不重要、不紧急不重要。
- 支持新建、编辑、完成、恢复、删除事项。
- 支持拖动事项在四个象限之间移动。
- 事项字段包括：标题、详细说明、所属象限、计划日期、截止时间、提醒时间、预计用时、状态和排序序号。
- 已完成事项默认折叠，可按日期、象限、状态筛选。
- 逾期事项保留原计划日期，并显示明确逾期状态，不自动改期。

### 4.3 日历

- 月视图为第一版默认日历模式。
- 每个日期显示任务数量和状态点，点击日期后查看当天事项与复盘。
- 支持从选定日期直接新建事项或补写复盘。
- 支持切换上月、下月和返回今天。

### 4.4 每日复盘

- 每个自然日只有一份复盘记录。
- 三个固定栏目：今日收获、待改进、明日重点。
- 输入停止 500 毫秒后自动保存，本地保存成功后显示轻量状态提示。
- 历史复盘可通过日历查看和修改。

### 4.5 专注计时

- 可从某个事项启动、暂停和结束专注计时。
- 专注记录关联事项和日期，用于计算“专注时长”。
- 软件意外退出时保留未结束计时状态；下次启动提示用户继续或结束。
- 第一版不提供番茄钟、白噪音或复杂专注方案。

### 4.6 提醒与桌面能力

- 事项可设置一次性提醒，到时发送 Windows 系统通知。
- 通知支持“打开事项”和“稍后 10 分钟提醒”。
- 支持最小化到系统托盘；托盘菜单提供显示主窗口、快速添加、暂停提醒和退出。
- 是否开机启动由用户在设置中选择，默认关闭。

### 4.7 数据备份与恢复

- 数据库自动保存在 Windows 用户应用数据目录。
- 支持手动导出完整备份文件，包含任务、复盘、专注记录、设置和数据库版本。
- 恢复前自动创建当前数据快照；恢复失败时自动回滚。
- 第一版不提供云同步和多人协作。

## 5. 五套工作日皮肤

所有皮肤共用相同组件、字号、间距、边框宽度和交互状态。皮肤只通过设计令牌切换。

| 工作日 | 皮肤名称 | 情绪定位 | 关键颜色 |
| --- | --- | --- | --- |
| 周一 | 冷启动 | 低电量、克制、慢慢进入工作 | 雾白 `#F1F3F4`、石板蓝 `#526B80`、琥珀 `#C49A55` |
| 周二 | 进入状态 | 理性、稳定、专注推进 | 冰白 `#F4F7FA`、钴蓝 `#245E9B`、青灰 `#67A6B8` |
| 周三 | 周中续航 | 疲惫但坚持、安静内省 | 珍珠灰 `#F4F1F5`、灰紫 `#63506A`、银灰 `#B5ADB8` |
| 周四 | 曙光将至 | 恢复能量、看到周末 | 矿物白 `#F3F6F2`、墨绿 `#2D6F65`、黄铜 `#A58A58` |
| 周五 | 准点下班 | 温暖、轻松、有释放感 | 暖象牙 `#F7F3EC`、陶土橙 `#B45F48`、香槟 `#B99A62` |

自动规则：周一到周五按系统本地日期自动使用对应皮肤；用户可在设置中临时手动选择，手动选择可设为“仅今天”或“保持使用”。周六、周日默认沿用用户最后一次手动选择；若从未手动选择，则使用中性的周二皮肤。应用不新增第六套周末皮肤。

## 6. 页面与组件结构

### 主导航

- 今日工作台
- 今天
- 四象限
- 日历
- 复盘
- 统计
- 设置

### 关键组件边界

- `TaskService`：任务增删改查、排序、象限移动和状态变化。
- `ReviewService`：每日复盘读取、自动保存和历史查询。
- `FocusService`：专注会话状态与时长统计。
- `ReminderService`：提醒计划、到期触发和稍后提醒。
- `ThemeService`：工作日自动皮肤、手动覆盖和设计令牌。
- `BackupService`：备份、校验、恢复和失败回滚。
- `Database`：SQLite 连接、事务、迁移和数据访问。
- Electron 主进程：窗口、托盘、通知、开机启动和安全 IPC。
- React 渲染进程：页面、组件、交互状态和无权限的界面逻辑。

## 7. 数据模型

### tasks

- `id`：UUID
- `title`：必填，1–120 字符
- `notes`：可选，最多 5000 字符
- `quadrant`：`urgent_important`、`important`、`urgent`、`neither`
- `planned_date`：本地日期 `YYYY-MM-DD`
- `due_at`：可选，本地日期时间
- `remind_at`：可选，本地日期时间
- `estimated_minutes`：可选正整数
- `status`：`active`、`completed`、`deleted`
- `sort_order`：同一列表内排序
- `completed_at`、`created_at`、`updated_at`

### daily_reviews

- `id`：UUID
- `review_date`：唯一的本地日期
- `wins`、`improvements`、`tomorrow_focus`
- `created_at`、`updated_at`

### focus_sessions

- `id`：UUID
- `task_id`：可选关联任务
- `started_at`、`ended_at`
- `duration_seconds`
- `state`：`running`、`paused`、`finished`

### reminders

- `id`、`task_id`、`scheduled_at`
- `status`：`pending`、`fired`、`snoozed`、`cancelled`
- `snooze_count`、`last_fired_at`

### settings

- 键值形式保存主题覆盖、开机启动、托盘行为、提醒开关和最近备份路径。

## 8. 核心数据流

1. 用户在渲染进程操作界面。
2. 经过类型安全的 IPC 调用 Electron 主进程服务。
3. 主进程在 SQLite 事务中写入数据。
4. 成功后返回更新后的领域对象；界面刷新相关视图和统计。
5. 提醒服务根据数据库中的待触发提醒重建内存计划；软件重启后不会丢失提醒。
6. 皮肤服务根据系统本地星期和用户覆盖设置生成主题令牌，所有页面统一消费这些令牌。

渲染进程不能直接访问 Node.js、文件系统或数据库。启用上下文隔离，所有 IPC 通道使用固定白名单和参数校验。

## 9. 异常与恢复策略

- 数据库写入失败：保持界面原值，显示可理解的错误提示并记录本地日志。
- 自动保存失败：保留输入内容，持续显示“未保存”，允许手动重试。
- 提醒时间已过：应用启动后检查；24 小时内的未触发提醒补发一次，更早的只标记为错过。
- 系统时间或时区变化：重新计算未触发提醒，不修改已经保存的本地计划日期。
- 数据库迁移失败：不进入主界面，保留原数据库并提供恢复或导出诊断信息。
- 恢复文件损坏：拒绝覆盖现有数据库，并说明校验失败。
- 单实例运行：再次打开软件时激活已有窗口，不启动第二个数据库实例。

## 10. 测试与验收

### 单元测试

- 工作日到皮肤的映射、手动覆盖和周末回退规则。
- 四象限移动、排序、完成和逾期计算。
- 今日统计与专注时长汇总。
- 复盘自动保存防抖逻辑。
- 提醒调度、稍后提醒和重启恢复。
- 备份校验和数据库迁移。

### 集成测试

- 创建事项后，工作台、四象限和日历同步出现。
- 拖动事项切换象限后，数据库和界面一致。
- 完成事项后，统计、日期标记和任务状态同步更新。
- 保存复盘并重启软件后，内容完整恢复。
- 启动专注计时、暂停、结束后，今日时长准确。
- 导出备份、修改数据、恢复备份后回到原状态。

### 端到端测试

- 首次启动与数据库初始化。
- 完整任务生命周期。
- 五种皮肤自动切换和手动覆盖。
- 日历选日、新建事项、写复盘的完整路径。
- 通知点击和稍后提醒路径。
- 托盘最小化、恢复和单实例行为。

### Windows 验收

- Windows 10/11 64 位安装与卸载正常。
- 在 100%、125%、150% 缩放下无文字遮挡、控件溢出或窗口错位。
- 1440×900 为设计基准，最低支持 1280×720。
- 正常数据量下冷启动目标小于 3 秒。
- 断网时所有核心功能正常，且应用不发起业务网络请求。
- 异常退出并重启后，已保存数据和正在计时的会话可恢复。

## 11. 第一版不包含

- 云同步、账号、多设备、团队共享。
- 手机端、网页端和 macOS 版本。
- AI 自动拆解任务或生成复盘。
- 周视图、日程邀请和第三方日历同步。
- 复杂重复任务、番茄钟、白噪音或社交排行榜。
- 插件市场和自定义皮肤编辑器。

## 12. 交付物

- 完整源代码和开发说明。
- 可运行的开发版。
- Windows 安装包构建配置。
- 自动化测试与测试报告。
- 五套工作日皮肤的统一设计令牌。
- 数据备份、恢复和升级说明。

## 13. 完成定义

只有同时满足以下条件才视为第一版完成：核心功能全部实现；自动化测试通过；关键流程完成 Windows 验收；五套皮肤按星期正确切换且布局一致；本地数据、提醒、备份和恢复验证通过；不存在阻断使用的已知缺陷。
