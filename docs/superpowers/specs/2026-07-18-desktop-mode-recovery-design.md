# 四象日志桌面模式恢复设计规格

日期：2026-07-18  
目标版本：0.3.3  
状态：待用户书面确认

## 1. 问题与目标

Windows 真桌面嵌入模式会把主窗口挂到桌面图标层之后。桌面图标层会拦截鼠标输入，因此嵌入窗口虽然可见，但窗口内部的“恢复窗口”、设置和事项编辑控件可能无法点击。

本次修复的目标是：无论主窗口是否还能接收鼠标输入，用户都能通过清晰、稳定的入口退出桌面模式并恢复全部编辑能力。

## 2. 采用方案

保留现有 WorkerW 真桌面嵌入能力，并增加独立的桌面恢复控制条。控制条是普通顶层小窗口，不嵌入 WorkerW，因此不会被桌面图标层挡住。

同时把托盘、通知、第二实例和快捷键等外部入口统一为“先退出桌面模式，再显示并聚焦主窗口”，避免出现只显示进程或仍停留在桌面层后的情况。

不采用以下方案：

- 只依赖托盘恢复：入口不够直观，托盘图标可能被 Windows 折叠。
- 取消 WorkerW、改成普通底层窗口：虽然可交互，但会削弱真正嵌入桌面的效果。

## 3. 桌面恢复控制条

- 成功进入桌面模式后，在屏幕右下角显示一个紧凑控制条。
- 控制条显示“四象日志已嵌入桌面”和主按钮“恢复并编辑”。
- 点击“恢复并编辑”后，先解除 WorkerW 挂载，再恢复窗口边框、任务栏显示、100% 普通窗口透明度、原窗口位置和焦点。
- 恢复成功后自动关闭控制条。
- 嵌入失败、应用退出或主窗口被销毁时，控制条必须关闭。
- 控制条不加入任务栏、不抢占启动焦点、不可重复创建，并始终保持在普通桌面窗口之上。
- 多显示器场景下，控制条显示在主窗口当前所在显示器的工作区右下角；工作区变化时重新校正位置。
- 控制条不承载设置或业务编辑功能，只负责安全恢复，避免形成第二套界面。

## 4. 统一恢复入口

主进程提供唯一的 `restoreAndShow()` 流程，以下入口全部调用它：

- 控制条“恢复并编辑”。
- 托盘单击“显示四象日志”。
- 托盘双击。
- 托盘“恢复主窗口”。
- 托盘“快速添加事项”（恢复完成后再打开快速添加）。
- 提醒通知点击。
- 已有实例运行时再次启动程序。
- 全局快捷键 `Ctrl+Alt+J`。

`restoreAndShow()` 必须可重复调用：普通窗口状态下直接显示并聚焦；桌面模式下只执行一次解除挂载。多个入口同时触发时复用同一个恢复任务，不重复修改窗口样式。

## 5. 快捷键与生命周期

- 应用启动成功后注册 `Ctrl+Alt+J`，用于紧急恢复并显示主窗口。
- 如果快捷键已被其他程序占用，应用继续正常运行，并记录失败；托盘和控制条仍可恢复。
- 应用退出时注销快捷键。
- Windows Explorer 重启、原生辅助程序异常或解除挂载失败时，仍强制恢复 Electron 普通窗口样式并显示主窗口，同时向用户提示“桌面嵌入已退出，请重试”。

## 6. 错误处理

- 控制条创建失败：立即退出桌面模式并恢复主窗口，不允许进入没有外部恢复入口的嵌入状态。
- 原生解除挂载失败：继续执行本地窗口样式、任务栏、透明度、位置和显示恢复；错误写入日志并提示用户。
- 快速添加和通知点击：等待恢复流程完成后再打开目标界面，避免消息发送给不可交互的窗口。
- 连续点击恢复按钮：按钮进入“正在恢复”状态，忽略重复操作。
- 关闭控制条不等同于关闭应用；控制条不提供单独的系统关闭按钮。

## 7. 范围与兼容性

- 不改变六套皮肤、主界面布局、事项数据、报告模板、月份切换、透明度数值范围及数据库结构。
- 不增加联网服务和后台遥测。
- 仅 Windows 启用 WorkerW 和恢复控制条；其他平台保持现有普通窗口行为。
- 现有用户设置和数据无需迁移。

## 8. 测试与验收

### 自动化测试

- 桌面模式成功后只创建一个恢复控制条。
- 控制条点击会调用统一恢复流程，成功后关闭。
- 托盘显示、双击、恢复、快速添加、通知点击、第二实例和快捷键均先恢复再执行后续动作。
- 普通窗口调用恢复流程不会重复解除挂载。
- 并发恢复只执行一次原生解除操作。
- 控制条创建失败会回滚为可见普通窗口。
- 快捷键注册失败不影响应用启动，退出时执行注销。
- 解除挂载失败仍恢复边框、任务栏、透明度和可见性。

### Windows 打包验证

- 运行完整单元测试、类型检查和生产构建。
- 在 Windows CI 中编译原生桌面辅助程序并生成绿色版。
- 验证绿色版目录结构、主程序启动和文件校验值。
- 自动验证原生辅助程序的参数校验、状态查询及错误返回。
- 记录一份人工实机验收清单：进入嵌入模式后分别使用控制条、托盘、通知和 `Ctrl+Alt+J` 恢复；每次恢复后均可进入设置并编辑事项。

## 9. 完成标准

满足以下条件才可发布 0.3.3：

1. 嵌入桌面后始终存在至少一个无需点击嵌入窗口即可使用的恢复入口。
2. 从所有恢复入口退出后，主窗口可见、可聚焦、设置可打开、事项可编辑。
3. 自动化测试、完整源码验证、Windows 构建和绿色版完整性检查全部通过。
4. 发布说明明确列出控制条、统一恢复入口和紧急快捷键。
