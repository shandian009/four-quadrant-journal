# 四象日志六套工作日皮肤设计规格

日期：2026-07-11
状态：用户已确认设计方向，待规格复核

## 1. 目标

删除当前五套工作日配色，保持现有布局、功能、文字层级和交互逻辑不变，替换为六套参考图风格皮肤。皮肤按周一至周六自动匹配，周日跟随周六；设置页继续支持当日覆盖和长期覆盖。

本次仅调整视觉主题系统及其自动映射，不调整事项、日历、复盘、统计、专注、提醒、数据库和备份逻辑。

## 2. 采用方案

采用“统一布局 + 全局主题令牌 + 分区渐变令牌”。

- 全部页面共享同一组件树和尺寸体系。
- 每套皮肤提供画布、侧栏、卡片、弱化卡片、文字、边框、强调色和微强调色。
- 今日事项、月历、今日概览、今日复盘可使用同主题下不同方向或明度的轻微渐变。
- 禁止为不同皮肤复制页面组件或改变功能入口。
- 深色主题不使用纯黑大面积压底；浅色主题避免大面积高饱和色。

## 3. 自动匹配

| 星期 | 主题标识 | 界面标签 | 情绪定位 |
|---|---|---|---|
| 周一 | `monday` | 周一 · 冷启动 | 深海蓝科技，理性进入工作 |
| 周二 | `tuesday` | 周二 · 渐入状态 | 极光蓝清新渐变，效率提升 |
| 周三 | `wednesday` | 周三 · 舒缓续航 | 薄荷绿自然，降低周中疲劳 |
| 周四 | `thursday` | 周四 · 沉稳推进 | 墨绿治愈，保持稳定专注 |
| 周五 | `friday` | 周五 · 冲刺收官 | 炭黑橙极简，强化完成感 |
| 周六 | `saturday` | 周六 · 松弛复盘 | 青灰蓝沉稳，整理一周 |
| 周日 | `saturday` | 周六 · 松弛复盘 | 跟随周六，休息与轻量规划 |

手动覆盖仍使用现有 `today` 与 `persistent` 两种模式。旧设置中 `monday` 至 `friday` 的标识保持兼容，但对应视觉会替换为新方案；新增 `saturday`。

## 5. 分区渐变规则

- 应用外层画布使用主题画布渐变。
- 侧栏使用独立侧栏渐变，当前导航项使用主强调色到辅助强调色的低对比渐变。
- 今日事项以主题主色为主；月历向冷色或中性色偏移；今日概览允许使用辅助强调色；今日复盘三栏分别使用主强调、辅助强调和微强调。
- 渐变只用于背景、选中态和进度线，不应用于长篇文字。
- 边框保持 1px，深色主题允许轻微内发光但不使用强烈霓虹。
- 阴影保持柔和，禁止改变卡片尺寸、间距和排列。

## 6. 设置与兼容

- 设置页主题选择项由五套更新为六套。
- 不迁移或删除数据库中的主题设置键。
- 已保存的 `monday` 至 `friday` 覆盖继续有效，只显示新配色。
- 无效主题值继续由现有校验或默认逻辑处理。
- 周日自动解析为 `saturday`，但手动当日或长期覆盖优先级不变。

## 7. 可访问性

- 正文与卡片背景对比度目标不低于 WCAG AA 的 4.5:1。
- 次文字目标不低于 3:1，必要时优先调整次文字明度。
- 危险操作继续使用独立危险色，不因主题主色改变而失去识别度。
- 焦点、选中和进度状态不能只依赖颜色，还保留边框、图标或形状差异。

## 8. 验证标准

- 周一至周六分别解析到六个主题标识，周日解析到 `saturday`。
- 六套主题拥有完全一致的令牌字段集合。
- 设置页能够选择并持久化 `saturday`。
- 手动当日覆盖和长期覆盖继续生效。
- 六套皮肤切换时页面区域、尺寸和功能入口不改变。
- 浅色和深色主题均能通过核心文字可读性检查。
- `npm run verify`、Windows生产构建和打包产物真实启动测试通过。

## 9. 非目标

- 不新增主题编辑器、颜色选择器或主题下载功能。
- 不更改窗口标题栏、菜单栏和系统原生按钮。
- 不调整任务、复盘、统计或提醒业务逻辑。
- 不为周日新增第七套皮肤。
